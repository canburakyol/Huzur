# Contributing to Huzur

First off, thank you for considering contributing to Huzur! It's people like you that make Huzur such a great tool.

## How Can I Contribute?

### Reporting Bugs
- Use the GitHub Issues tab.
- Describe the bug and provide steps to reproduce.
- Include environment details (OS, Screen size, etc.).

### Suggesting Enhancements
- Open a GitHub Issue with the tag `enhancement`.
- Describe the feature and why it would be useful.

### Pull Requests
1. Fork the repo and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. If you've changed APIs, update the documentation.
4. Ensure the test suite passes.
5. Make sure your code follows the existing style.

## Style Guide
- Follow the project's ESLint and Prettier configurations.
- Write clear, descriptive commit messages using [Conventional Commits](https://www.conventionalcommits.org/).

## Questions?
Feel free to open an issue for any questions you might have.
