# Skill: Engineering Planning

A systematic approach to breaking down complex tasks into executable plans.

## Design Phase (Brainstorming)
- Ask clarifying questions before writing code.
- Explore at least two alternative implementation paths.
- Present the design in digestable sections for validation.
- Save the design document to `_bmad/plans/`.

## Planning Phase
- Break work into bite-sized tasks (2-5 minutes each).
- Specify exact file paths for every task.
- Define clear verification steps for each task.
- Use `task.md` to track progress.

## Workflow Integration
- Always check for the existence of a plan before starting EXECUTION mode.
- Update the plan if unexpected complexity is discovered.
