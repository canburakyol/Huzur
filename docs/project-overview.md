# Project Overview: Huzur App

**Huzur App** is a comprehensive Islamic lifestyle application designed to assist users in their daily religious practices. Built with modern web technologies and wrapped as a mobile application, it offers a rich set of features including prayer times, Quran reading, Qibla direction, and daily inspirations.

## 🚀 Key Features

- **Prayer Times**: Accurate prayer times based on location.
- **Quran**: Full Quran text with translations and audio recitations.
- **Qibla Finder**: Tool to find the direction of Qibla.
- **Daily Content**: Daily verses, hadiths, and prayers.
- **Tracking**: User engagement tracking for daily tasks and prayers.
- **Family Mode**: Features designed for family engagement.
- **Multimedia**: Audio and visual content integration.

## 🛠 Technology Stack

The project leverages a robust modern stack to ensure performance and cross-platform compatibility:

| Component | Technology | Description |
|-----------|------------|-------------|
| **Frontend Framework** | [React 19](https://react.dev/) | Component-based UI library for building interactive interfaces. |
| **Build Tool** | [Vite 7](https://vitejs.dev/) | Next-generation frontend tooling for fast development and building. |
| **Mobile Runtime** | [Capacitor 8](https://capacitorjs.com/) | Cross-platform runtime to deploy web apps to iOS and Android. |
| **Backend / Auth** | [Firebase 12](https://firebase.google.com/) | Backend-as-a-Service for Auth, Firestore, Functions, and Hosting. |
| **Language** | [TypeScript](https://www.typescriptlang.org/) | Strongly typed JavaScript for safer code. |
| **State Management** | Context API | Built-in React state management for global application state. |
| **Maps** | [Leaflet](https://leafletjs.com/) | Interactive maps for Qibla and location features. |
| **Styling** | CSS / Tailwind | Modern CSS for responsive and clean UI design. |

## 🏗 Architecture Type

**Mobile Hybrid Monolith**

The application follows a **Monolithic** architecture within a hybrid mobile context.
- **Single Codebase**: The entire application logic resides in a single repository (`src/`).
- **Client-Side Rendering (CSR)**: The React app is bundled and served locally on the device via Capacitor.
- **Service-Based Logic**: Business logic is encapsulated in a dedicated Service Layer (`src/services/`) which interacts with Firebase and device APIs.

## 📂 Repository Structure

The project is structured as a standard Vite + Capacitor project:

- `src/`: Core application source code.
- `public/`: Static assets served directly.
- `android/`: Native Android project files generated by Capacitor.
- `functions/`: Firebase Cloud Functions for backend logic.
- `docs/`: Project documentation.
